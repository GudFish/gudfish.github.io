# 2.1 飞控固件烧录
本设计飞控采用pixhawk1，固件采用ArduSub stable-4.5.7。针对于飞控本身无固件的情况，固件烧录可参考[ArduSub官方手册](https://ardupilot.org/sub/docs/common-loading-firmware-onto-chibios-only-boards.html "load firmware")。

本文采用从源码编译烧录的方式，[下载源代码](https://ardupilot.org/dev/docs/where-to-get-the-code.html "download code")并[配置编译环境](https://ardupilot.org/dev/docs/building-the-code.html "setup the build environment")。
```shell
Tools/environment_install/install-prereqs-ubuntu.sh -y.   # Install some required packages
. ~/.profile    # Reload the path (log-out and log-in to make it permanent)
```
下载并配置完成后，具体如何编译和上传固件到板子可参考[该文档](https://github.com/ArduPilot/ardupilot/blob/master/BUILD.md "how to build")。
```shell
./waf configure --board Pixhawk1       # 配置板子为Pixhawk1
./waf --targets bin/ardusub --upload   # usb连接板子后，编译ardusub并上传
```
如需不同版本的固件，可在[ArduPilot固件仓库](https://firmware.ardupilot.org/ "firmeware")中找到所需的版本下载。如：Sub/stable-4.5.7/Pixhawk1/ardusub.apj 为本设计所采用版本，并可以通过QGC连接后手动烧录。

6) 软件源

### b. Jetson Orin NX SUPER
考虑到树莓派机载算力较小，后续可以使用Jetson模组，这里说明如何为Jetson刷系统，教程参考该[链接](https://zhuanlan.zhihu.com/p/1987840057252548826)

1) 所需软硬件
- SDK Manager [下载链接](https://developer.nvidia.com/sdk-manager#getting-started)
- USB-C数据线
- 19V 45W直流稳压供电

2) 下载并烧录系统镜像
在Host电脑中打开sdkmananger软件后，登陆NVIDIA账号，没有可以注册一个。采用短接线将FC REC与GND进行短接，以使Jetson进取刷机模式。随后使用USB-C连接Jetson与Host电脑，并为Jetson供电。
- STEP 01中，PRODUCT CATEGORY选择Jetson，如果USB-C连接正常，则会在Target Hardware中识别到Jetson型号。注意不用勾选Host Machine，这是为当前电脑装系统。**如果需要刷SUPER模式，则SDK VERSION需要选择JetPack 5.1.5版本**
- STEP 02中，只需要勾选Jetson Linux，其他组件建议在装好系统后手动安装
- STEP 03中，弹出Flash界面时，点击Skip，因为SUPER模式需要通过脚本手动Flash。首先关闭USB自动挂起，避免Flash连接断开
```shell
sudo sh -c "echo -1 > /sys/module/usbcore/parameters/autosuspend"
```
接着进入JetPack 5.1.5安装目录，
```shell
cd /home/$USER/nvidia/nvidia_sdk/JetPack_5.1.5_Linux_JETSON_ORIN_NX_TARGETS/Linux_for_Tegra
```
根据Jetson型号，输入Flash命令，参考[官方文档](https://docs.nvidia.com/jetson/archives/r35.6.1/DeveloperGuide/IN/QuickStart.html)，这里为Jetson Orin NX SUPER进行具有SUPER模式的Flash，存储设备为NVMe，
```shell
sudo ./tools/kernel_flash/l4t_initrd_flash.sh --external-device nvme0n1p1 \
  -c tools/kernel_flash/flash_l4t_external.xml -p "-c bootloader/t186ref/cfg/flash_t234_qspi.xml" \
  --showlogs --network usb0 jetson-orin-nano-devkit-super internal
```
等待Flash完成，关闭Jetson供电，取下FC REC与GND的短接头，插上DP显示器、鼠标和键盘，再次通电即可进入Jetson的Ubuntu安装引导，按步骤依次进行即可。

3) 配置Ubuntu源(Ubuntu 20.04)

```shell
sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak  # 备份源
sudo vim /etc/apt/sources.list                          # 编辑源
```
输入`i`进入编辑模式，将deb开头的注释掉，并粘贴添加以下清华源

deb-src http://mirrors.tuna.tsinghua.edu.cn/ubuntu-ports/ focal-backports main restricted universe multiverse

按`Esc`退出编辑模式，输入`:wq`，并按`Enter`保存退出，随后更新源，依次执行下面的命令，安装jtop与包含cuda、cudnn、docker、tensorrt等的nvidia-jetpack软件包

sudo pip3 install -U jetson-stats

安装完成后，重启，在命令行输入`jtop`，验证是否安装成功。

# 2.3 飞控与机载电脑通信
## (1) 飞控(pixhawk1,ArduSub stable-4.5.7)
pixhawk1的micro USB接口和TELEM2端口均可以使用，这里直接使用USB转micro USB连接飞控和树莓派。需要连接地面站后设置串口参数

**SERIAL0_PROTOCOL** 设置为MAVLink2

**SERIAL0_BAUD** 建议设置为921600，飞控和(rpi)相距很近，速率高降低延迟

如果使用TELEM2端口，则上述应设置**SERIAL2_**的相关参数

## (2) 机载电脑(Ubuntu20.04)
### a. 树莓派3B+
树莓派利用USB接口更加稳定。前面测试了UART串口【RXD(GPIO 15, Pin 10)、TXD(GPIO 14, Pin 8)、GND (Header Pin 14)】，发现同时上电树莓派和pixhawk后，串口接受大量数据导致树莓派无法正常启动，因此后续更改为直接使用树莓派USB接口

### b. Jetson Orin NX SUPER
首先为当前用户赋予读取串口权限，将当前用户添加到`dialout`用户组，这是Ubuntu下访问串口设备的标准做法
```shell
sudo usermod -a -G dialout $USER
```
执行后，需要注销并重新登录，或重启系统才能使权限生效。随后插入USB连接飞控后，终端输入`ls /dev/ttyACM* /dev/ttyUSB*`查看端口名，如`/dev/ttyACM0`。
执行以下命令即可连接飞控
```shell
roslaunch mavros apm.launch fcu_url:=/dev/ttyACM0:921600
```

# 2.4 WaterLinked DVL A50连接到机载电脑
由于上电后机器人可能未下水，因此需要在控制器上电后，手动为DVL通电，因此利用飞控上的AUX端口作为开关，控制MOSFET通断来手动触发DVL的供电。选用Pixhawk1的AUX5端口，在QGC中

**SERVO13_FUNCTION** 设置为-1，这是将AUX5引脚释放，使其能作为通用输入输出引脚GPIO

**RELAY_PIN** 设置为AUX5，这个参数指定了第一个继电器由哪个物理引脚来控制，54对应的是AUX5

为实现对AUX5，即MOSFET通断控制，有两种方式，第一种通过手柄控制，在QGC的"游戏手柄/按钮分配"中，设置9通道为relay_1_toggle，后续按遥控器9通道即可切换AUX5输出状态；
第二种是在QGC软件中点击触发，为防止遥控器误触，选用该方式：在QGC的"Application Settings/Fly View"中找到MAVLink Actions，查看下方小字给出的json文件路径，创建**FlyViewActions.json**并放置在该路径下，随后重启QGC后，在同样位置的Fly View Actions中选择创建的json文件，之后，可在Fly View主页面左侧的Actions中鼠标点击相应的命令手动控制DVL通断。
```json
{
  "version": 1,
  "fileType": "MavlinkActions",
  "actions": [
    {
      "label": "DVL ON - AUX5",
      "description": "Open AUX5 Relay",
      "mavCmd": 181,
      "param1": 0,
      "param2": 1
    },
    {
      "label": "DVL OFF - AUX5",
      "description": "Close AUX5 Relay",
      "mavCmd": 181,
      "param1": 0,
      "param2": 0
    }
  ]
}
```

# 2.5 机载电脑/飞控与地面站的通信
开发了无线与有线两种方式，在近水面使用时，可以采用无线的方式，当潜入水下时，只能采用有线的方式控制。
## 无线方式
飞控插入数传电台，机载电脑插入WiFi模块，两者天线通过功分器连接，公用一根外部天线。这样，地面站通过局域网即可访问机载电脑，QGC通过数传模块可以访问飞控。

## 有线方式
机载电脑通过网口和电力载波板实现有线通信，本质还是同一局域网下的有线连接，其中机载电脑通过mavlink-router实现对飞控的数据的路由，一路由局域网UDP转发至地面站，一路由机载电脑的mavros接收用作机载电脑的ros程序使用。
1) 安装mavlink-router
- 确保安装软件包
```shell
sudo apt install git ninja-build pkg-config gcc g++ systemd  # 确保安装软件包
sudo pip3 install meson==0.63  # 单独安装meson（由于需要版本高于0.55）
```

- 获取源码
```shell
git clone https://github.com/mavlink-router/mavlink-router.git
cd mavlink-router
git submodule update --init --recursive
```
如果由于网络问题最后一条命令执行卡住，可以手动拉取submodule
```shell
cd ~/mavlink-router/modules
git clone https://github.com/mavlink/c_library_v2.git mavlink_c_library_v2
cd ~/mavlink-router
git submodule update --init --recursive  # 使git识别子模块已经就绪
```

- 编译并安装
```shell
meson setup build .
ninja -C build
sudo ninja -C build install
```

- 测试mavlink-router，先单次测试`sudo mavlink-routerd -e 192.168.2.1:14550 -e 127.0.0.1:14551 /dev/ttyACM0:921600`，将ttyACM0端口以921600识别，并路由为两个UDP，一个192.168.2.1:14550为地面站ip，另一个127.0.0.1:14551用于mavros的fcu_url读取。运行后测试QGC中的UDP连接，以及在机载电脑上进行mavros连接`roslaunch mavros apm.launch fcu_url:=127.0.0.1:14551`，如果两个链路通信无问题，则安装成功。

2) 设置系统服务
- 现在将上述测试命令封装为ubuntu系统服务，方便手动开启
```shell
sudo nano /etc/systemd/system/mavlink-router.service  # 创建编辑服务文件
```

- 粘贴以下内容
```ini
[Unit]
Description=MAVLink Router Service for UART to UDP
After=network.target

[Service]
Type=simple
User=root
Group=root
ExecStart=/usr/bin/mavlink-routerd -e 192.168.2.1:14550 -e 127.0.0.1:14551 /dev/ttyACM0:921600
Restart=on-failure
RestartSec=5
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
```

- 重新加载服务
```shell
sudo systemctl daemon-reload  # 让systemd重新加载所有服务配置文件，使新服务生效
```

- mavlink-router服务管理，后续可以启动服务，以用于路由USB端口连接的飞控mavlink信息
```shell
sudo systemctl start mavlink-router.service    # 启动
sudo systemctl stop mavlink-router.service     # 停止
sudo systemctl restart mavlink-router.service  # 重启
sudo systemctl status mavlink-router.service   # 查看状态
sudo systemctl enable mavlink-router.service   # 开机自启
sudo systemctl disable mavlink-router.service  # 关闭开机自启
sudo journalctl -u mavlink-router.service -f   # 查看实时日志
```
