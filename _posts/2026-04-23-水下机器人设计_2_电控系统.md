# 树莓派与pixhawk通信
## 树莓派
树莓派利用UART串口【RXD(GPIO 15, Pin 10)、TXD(GPIO 14, Pin 8)、GND (Header Pin 14)。硬件连接后，需要在树莓派的Ubuntu系统中进行配置，以启用串口。

(1)启用硬件串口并禁用蓝牙

树莓派的PL011串口（/dev/ttyAMA0）默认被蓝牙占用，需要释放出来给GPIO使用。

编辑配置文件：打开终端，编辑配置文件`sudo nano /boot/firmware/usercfg.txt`。如果文件不存在，则创建它。

添加以下两行：
```shell
enable_uart=1
dtoverlay=disable-bt
```
(2)禁用串口登录服务

该操作避免启动时pixhawk向树莓派发送大量mavlink数据，导致树莓派无法启动
```shell
sudo systemctl stop serial-getty@ttyAMA0.service
sudo systemctl disable serial-getty@ttyAMA0.service
```
(3)修改`/boot/firmware/cmdline.txt`

此文件定义了Linux内核的启动参数。需要确保console参数不占用硬件串口。

编辑文件：`sudo nano /boot/firmware/cmdline.txt`

找到并删除其中的console=ttyAMA0,115200或console=serial0,115200。例如，将console=tty1 console=ttyAMA0,115200 ...修改为console=tty1 ...。

保存修改。`Ctrl+o`，`Enter`，`Ctrl+x`

(4)将用户加入dialout组

将当前用户添加至dialout组，以获得对串口设备的读写权限。执行以下命令，完成后需要重新登录或再次重启以生效
```shell
sudo usermod -a -G dialout $USER
```
## 飞控(pixhawk1)
pixhawk1的TELEM2端口可直接连接树莓派，需要连接地面站后设置串口参数

**SERIAL2_PROTOCOL** 设置为MAVLink2

**SERIAL2_BAUD** 建议设置为921600，飞控和伴随计算机(rpi)相距很近，速率高降低延迟
